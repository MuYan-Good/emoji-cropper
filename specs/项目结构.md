# 项目结构：表情包雪碧图裁剪工具

## 1. 架构模式

采用**分层架构（Layered Architecture）**，按职责划分目录层级。本项目是轻量级单页应用，不需要复杂的特性模块拆分，清晰的分层足以保证高内聚低耦合。

```
src/
├── components/    # UI 组件层 — 纯展示，不包含业务逻辑
├── hooks/         # 自定义 Hooks — 封装可复用的状态逻辑
├── utils/         # 工具函数层 — 纯函数，处理图片裁剪、GIF、ZIP 等
├── types/         # 类型定义层 — TypeScript 类型和接口
└── assets/        # 静态资源 — 图标、字体等
```

## 3. 目录职责说明

### 3.1 `components/` — UI 组件层

所有 React 组件放在此目录，遵循**单一职责原则**：

| 组件 | 职责 | 关键交互 |
|------|------|----------|
| `ImageUploader` | 拖拽/点击上传区域 | 调用 `useImageUpload` 处理文件 |
| `GridPreview` | 在原图上叠加网格分割线 | 接收行列数参数，实时渲染 |
| `CropControls` | 行列数输入 + 输出尺寸选择 | 表单控件，触发裁剪操作 |
| `EmojiCard` | 单个表情的缩略图 + 可编辑名称 | 点击名称进入编辑模式 |
| `EmojiGrid` | 卡片网格布局容器 | 渲染 `EmojiCard` 列表 |
| `PreviewModal` | 点击卡片后的大图预览弹窗 | 支持 GIF 动图播放预览 |
| `ExportPanel` | 导出操作按钮区域 | 单张下载 + 批量 ZIP 打包 |
| `Header` | 页面标题和简介 | 纯展示 |

### 3.2 `hooks/` — 自定义 Hooks 层

封装可复用的状态逻辑，将业务逻辑从组件中抽离：

| Hook | 职责 | 依赖 |
|------|------|------|
| `useImageUpload` | 文件校验（类型、大小）、读取为 Image/ArrayBuffer | `utils/fileHelper` |
| `useSpriteCrop` | 根据行列数裁剪雪碧图，调用 PNG/GIF 处理器 | `utils/imageCropper`, `utils/gifProcessor` |
| `useEmojiGrid` | 管理裁剪结果列表、文件名编辑、选中状态 | — |
| `useExport` | 单张下载、批量 ZIP 打包 | `utils/zipExporter`, `utils/fileHelper` |

### 3.3 `utils/` — 工具函数层

纯函数，不依赖 React，便于单元测试：

| 模块 | 职责 | 核心依赖 |
|------|------|----------|
| `imageCropper` | PNG 图片的 Canvas 裁剪与缩放 | Canvas API |
| `gifProcessor` | GIF 解析为帧序列 → 逐帧裁剪 → 重新编码为 GIF | gifuct-js |
| `zipExporter` | 将多个文件打包为 ZIP 并触发下载 | JSZip, FileSaver.js |
| `fileHelper` | 文件名生成（`emoji_01.png`）、格式校验、大小格式化 | — |
| `constants` | 默认输出尺寸、文件名前缀、支持格式列表等 | — |

### 3.4 `types/` — 类型定义层

集中管理 TypeScript 类型，确保类型一致性：

| 文件 | 核心类型 |
|------|----------|
| `emoji.ts` | `EmojiItem`（裁剪结果）、`CropResult`（裁剪输出） |
| `image.ts` | `ImageFile`（上传的图片）、`GridConfig`（行列数配置） |
| `export.ts` | `ExportOptions`（导出参数）、`ExportFormat`（PNG/GIF） |

## 4. 数据流设计

```
用户上传图片
    │
    ▼
ImageUploader ──→ useImageUpload ──→ 校验 + 读取图片
    │
    ▼
GridPreview + CropControls ──→ 用户设置行列数 + 输出尺寸
    │
    ▼
useSpriteCrop ──→ imageCropper / gifProcessor ──→ 裁剪 + 缩放
    │
    ▼
useEmojiGrid ──→ EmojiGrid (EmojiCard[]) ──→ 卡片展示 + 命名编辑
    │
    ▼
useExport ──→ zipExporter ──→ 单张下载 / ZIP 批量打包
```

## 5. 设计原则

1. **组件纯化**：`components/` 中的组件只负责 UI 渲染和用户交互，业务逻辑通过 hooks 注入
2. **工具函数无副作用**：`utils/` 中的函数均为纯函数，输入相同则输出相同，便于测试
3. **类型集中管理**：所有共享类型定义在 `types/` 中，避免循环依赖
4. **单向数据流**：数据从 hooks 流向 components，用户操作通过回调函数回传给 hooks
5. **AI 接口预留**：`utils/` 中预留 `aiNamer.ts` 接口文件位置，后续迭代时添加
