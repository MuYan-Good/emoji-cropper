# 产品概述：表情包雪碧图裁剪工具

## 1. 产品愿景

打造一款**零门槛、开箱即用**的 Web 端表情包裁剪工具，帮助表情包创作者从雪碧图（Sprite Sheet）中一键裁剪出单个表情，快速命名并导出为符合微信表情开放平台规范的文件包，大幅缩短从创作到提交的工作流。

## 2. 核心价值

| 维度 | 说明 |
|------|------|
| **效率提升** | 将手动裁剪 + 重命名 + 缩放的流程从数十分钟缩短到几秒 |
| **零安装成本** | 纯 Web 工具，浏览器打开即用，无需安装任何软件 |
| **平台适配** | 内置微信表情开放平台规范（240×240px），裁剪即合规 |
| **灵活命名** | 自动序号 + 手动编辑，满足不同命名习惯 |

## 3. 目标用户

| 用户画像 | 特征描述 |
|----------|----------|
| **表情包创作者** | 设计师或绘画爱好者，制作原创表情包并希望上架微信表情开放平台 |
| **表情包搬运/整理者** | 收集整理表情包素材，需要批量裁剪和标准化处理 |
| **小型工作室** | 批量制作表情包套装，需要高效的裁剪和导出流程 |

## 4. 核心功能板块

### 4.1 图片上传与解析

- 支持拖拽或点击上传雪碧图（PNG/GIF）
- 用户手动输入行列数（如 4 行 4 列）
- 实时预览网格分割线，确认裁剪区域

### 4.2 一键裁剪

- 根据行列数自动等分切割雪碧图
- 裁剪后默认缩放到 240×240px（微信平台标准尺寸）
- 支持用户自定义输出尺寸（如 120×120px、512×512px 等）

### 4.3 卡片网格预览与命名

- 裁剪结果以卡片网格形式展示，每张卡片包含缩略图 + 可编辑文件名
- 默认按序号自动命名（如 `emoji_01.png`、`emoji_02.png` ...）
- 点击文件名即可原地编辑，支持中文/英文/数字命名

### 4.4 导出与下载

- **单张下载**：点击卡片可预览大图并单独下载
- **批量打包**：一键将所有裁剪结果打包为 ZIP 文件下载
- ZIP 内文件结构清晰，可直接用于微信表情开放平台提交

### 4.5 AI 命名接口（预留）

- 预留 AI 识别接口，后续迭代可接入视觉大模型自动识别表情内容并生成中文名称
- 接口设计为可插拔式，不影响当前版本功能

## 5. 典型使用场景

### 场景一：原创表情包上架微信平台

> 创作者在画板软件中完成一套 16 个表情的设计，导出为一张 4×4 的雪碧图。打开本工具 → 上传图片 → 输入 4 行 4 列 → 一键裁剪 → 逐个修改名称（如"开心"、"生气"）→ 批量下载 ZIP → 提交微信表情开放平台。

### 场景二：素材整理与标准化

> 用户收集到一批不同尺寸的表情包合集图，需要统一裁剪为 240×240px 的标准尺寸。上传 → 设置行列数 → 选择输出尺寸 240×240 → 批量下载。

## 6. 非功能性约束

| 约束项 | 要求 |
|--------|------|
| **运行环境** | 现代浏览器（Chrome、Edge、Firefox、Safari），无需安装 |
| **数据处理** | 所有图片处理在浏览器端完成，不上传到任何服务器，保护用户隐私 |
| **文件格式** | 支持 PNG、GIF 输入；输出保持原格式（PNG 输出 PNG，GIF 输出 GIF） |
| **性能要求** | 16 个表情的裁剪 + 缩放应在 2 秒内完成 |
| **响应式** | 支持桌面端浏览器使用，移动端暂不作为优先级 |

## 7. 项目边界

### 本期包含（V1.0）

- 雪碧图上传与行列数手动输入
- 自动裁剪 + 缩放到指定尺寸
- 卡片网格预览 + 手动编辑文件名
- 单张下载 + ZIP 批量打包下载
- 面向微信表情开放平台规范（240×240px）

### 本期不包含（后续迭代）

- AI 自动识别表情内容并命名
- 自动检测行列数（智能网格识别）
- 表情包套装管理（多套表情包项目管理）
- 拖拽排序表情顺序
- 其他平台适配（LINE、Telegram 等）
- 移动端适配

## 8. 验收标准

| 编号 | 验收条件 |
|------|----------|
| AC-01 | 用户可上传 PNG/GIF 格式的雪碧图，图片大小支持至 10MB |
| AC-02 | 用户输入行列数后，页面实时显示网格分割线预览 |
| AC-03 | 点击"一键裁剪"后，正确按行列数等分切割图片 |
| AC-04 | 裁剪后默认缩放到 240×240px，用户可选择其他尺寸 |
| AC-05 | 裁剪结果以卡片网格展示，每张卡片显示缩略图和可编辑文件名 |
| AC-06 | 默认文件名格式为 `emoji_XX.png`（XX 为两位序号），用户可点击修改 |
| AC-07 | 支持单张表情预览和下载 |
| AC-08 | 支持一键 ZIP 打包下载所有裁剪结果 |
| AC-09 | GIF 格式输入时，裁剪后输出仍为 GIF 格式（保持动画） |
| AC-10 | 所有图片处理在浏览器端完成，不依赖后端服务 |
