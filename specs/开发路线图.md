# 开发路线图：表情包雪碧图裁剪工具

## 模块依赖分析

### 依赖关系图

```
types/ (地基 — 无依赖)
  ↑
utils/constants.ts (依赖 types/)
  ↑
utils/fileHelper.ts (依赖 types/, constants.ts)
  ↑
utils/imageCropper.ts (依赖 types/, constants.ts)
utils/gifProcessor.ts (依赖 types/, constants.ts)
  ↑
utils/zipExporter.ts (依赖 types/, fileHelper.ts)
  ↑
hooks/ (依赖 utils/, types/)
  ↑
components/ (依赖 hooks/, types/)
  ↑
App.tsx (编排所有组件和 hooks)
```

### 模块分类

| 类别 | 模块 | 说明 |
|------|------|------|
| **地基** | `types/`、`utils/constants.ts`、`utils/fileHelper.ts` | 无外部依赖，其他所有模块的基础 |
| **核心引擎** | `utils/imageCropper.ts`、`utils/gifProcessor.ts` | 项目核心能力，裁剪和缩放 |
| **业务逻辑** | `hooks/useImageUpload.ts`、`hooks/useSpriteCrop.ts`、`hooks/useEmojiGrid.ts`、`hooks/useExport.ts` | 连接 UI 和引擎 |
| **导出能力** | `utils/zipExporter.ts` | 依赖 fileHelper，为 hooks 层服务 |
| **UI 层** | `components/` 所有组件 | 依赖 hooks，最终用户交互 |
| **编排层** | `App.tsx` | 组装所有组件，管理全局状态 |

## Milestone 1: 项目初始化与地基搭建

> 目标：项目可运行，类型系统和基础工具函数就位

- [ ] **1.1 项目初始化**
  - 使用 Vite 创建 React + TypeScript 项目
  - 配置 Tailwind CSS 4、ESLint、Prettier
  - 安装所有生产依赖和开发依赖
  - 创建目录结构（components/、hooks/、utils/、types/）
  - 验证：`pnpm dev` 可正常启动

- [ ] **1.2 类型定义（types/）**
  - `types/emoji.ts` — EmojiItem、CropResult
  - `types/image.ts` — ImageFile、GridConfig、ImageType
  - `types/export.ts` — ExportOptions、ExportFormat

- [ ] **1.3 常量与辅助函数（utils/）**
  - `utils/constants.ts` — DEFAULT_OUTPUT_SIZE、MAX_FILE_SIZE、SUPPORTED_TYPES、FILE_NAME_PREFIX
  - `utils/fileHelper.ts` — validateFile()、generateFileName()、formatFileSize()、getImageType()

## Milestone 2: 核心裁剪引擎（PNG）

> 目标：PNG 雪碧图可正确裁剪并缩放到指定尺寸

- [ ] **2.1 PNG 裁剪引擎（utils/imageCropper.ts）**
  - cropImage() — 从雪碧图中裁剪指定区域的单张图片
  - resizeImage() — 将图片缩放到指定尺寸
  - cropSprite() — 按行列数批量裁剪雪碧图
  - 单元测试：验证裁剪位置、缩放尺寸、输出格式

- [ ] **2.2 图片上传 Hook（hooks/useImageUpload.ts）**
  - 文件校验（类型、大小）
  - 读取为 HTMLImageElement
  - 错误处理和状态管理

- [ ] **2.3 裁剪 Hook（hooks/useSpriteCrop.ts）**
  - 接收图片和 GridConfig，调用 imageCropper 裁剪
  - 管理裁剪状态（loading、error、results）

- [ ] **2.4 上传与裁剪 UI（components/）**
  - `Header.tsx` — 页面标题
  - `ImageUploader.tsx` — 拖拽/点击上传区域
  - `CropControls.tsx` — 行列数输入 + 输出尺寸选择
  - `GridPreview.tsx` — 原图 + 网格分割线预览
  - `App.tsx` — 基础页面布局，串联上传 → 设置 → 裁剪流程
  - 验证：上传 PNG 雪碧图 → 输入行列数 → 点击裁剪 → 控制台输出裁剪结果

## Milestone 4: 导出与下载

> 目标：支持单张下载和 ZIP 批量打包下载

- [ ] **4.1 ZIP 打包工具（utils/zipExporter.ts）**
  - createZipFromEmojis() — 将多个表情打包为 ZIP
  - downloadSingle() — 单张图片下载

- [ ] **4.2 导出 Hook（hooks/useExport.ts）**
  - 单张下载逻辑
  - 批量 ZIP 打包逻辑
  - 下载状态管理

- [ ] **4.3 导出 UI（components/）**
  - `ExportPanel.tsx` — 批量打包下载按钮
  - `PreviewModal.tsx` — 点击卡片大图预览 + 单张下载
  - 验证：单张下载 PNG、批量 ZIP 打包下载

---

## Milestone 5: GIF 动图支持

> 目标：GIF 雪碧图可正确裁剪并保持动画效果

- [ ] **5.1 GIF 处理引擎（utils/gifProcessor.ts）**
  - parseGif() — 解析 GIF 为帧序列
  - cropGifFrame() — 裁剪单帧
  - encodeGif() — 将裁剪后的帧序列编码为 GIF
  - cropGifSprite() — 按行列数批量裁剪 GIF 雪碧图
  - 单元测试：验证帧数保持、延迟时间正确

- [ ] **5.2 裁剪 Hook 适配（hooks/useSpriteCrop.ts）**
  - 根据 ImageType 自动选择 PNG 或 GIF 处理路径
  - GIF 裁剪进度反馈

- [ ] **5.3 UI 适配**
  - `EmojiCard.tsx` — GIF 缩略图显示动画预览
  - `PreviewModal.tsx` — GIF 大图动画播放
  - 验证：上传 GIF → 裁剪 → 卡片显示动画 → 下载 GIF 保持动画

## Milestone 6: 完善与优化

> 目标：打磨体验，修复边界问题，准备发布

- [ ] **6.1 边界情况处理**

  - 重新上传 / 清空结果

- [ ] **6.3 构建与部署**
  - `pnpm build` 生产构建验证
  - 构建产物体积检查
  - 部署到静态托管平台（Vercel / Netlify / GitHub Pages）

## 开发顺序总览

```
M1: 项目初始化与地基搭建
 │   types/ → constants.ts → fileHelper.ts
 │
 ▼
M2: 核心裁剪引擎（PNG）
 │   imageCropper.ts → useImageUpload → useSpriteCrop → 上传+裁剪 UI
 │
 ▼
M3: 卡片网格与命名
 │   useEmojiGrid → EmojiCard → EmojiGrid
 │
 ▼
M4: 导出与下载
 │   zipExporter → useExport → ExportPanel → PreviewModal
 │
 ▼
M5: GIF 动图支持
 │   gifProcessor → useSpriteCrop 适配 → UI 适配
 │
 ▼
M6: 完善与优化
     边界处理 → UX 优化 → 构建部署
```

> **关键路径**：M1 → M2 → M3 → M4 是核心业务闭环，必须按顺序完成。M5（GIF）和 M6（优化）可并行推进或根据优先级调整。
