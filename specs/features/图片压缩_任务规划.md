## 切片3：压缩参数与实时预览

**完成标准**：用户可调整输出格式、质量滑块、输出尺寸，实时看到第一张图片的压缩预览效果和体积对比。

### 任务 3.1：实现 imageCompressor 核心函数 ⚠️

**通俗解释**：这是压缩功能的"引擎"，负责把图片按指定参数压缩输出。这是整个功能技术难度最高的部分。

**技术方案章节**：§3 核心压缩逻辑

**AC**：AC-C-05、AC-C-06、AC-C-07、AC-C-11

**验证标准**：
- `compressImage(pngFile, { format: 'jpeg', quality: 80, ... })` 返回 Blob，type 为 `image/jpeg`
- `compressImage(pngFile, { format: 'png', quality: 80, ... })` 返回 Blob，type 为 `image/png`
- `compressImage(jpegFile, { format: 'jpeg', quality: 50, ... })` 返回的 Blob 体积小于 quality=80 时的体积
- PNG→JPEG 时，透明区域填充白色（检查压缩后图片无透明像素）
- `compressImage(gifFile, ...)` 抛出"不支持 GIF 格式压缩"
- `calculateOutputSize(800, 600, 50, true)` 返回 `{ width: 400, height: 300 }`
- `calculateOutputSize(800, 600, 100, true)` 返回 `{ width: 800, height: 600 }`
- `calculateOutputSize(800, 600, 100, false, 200, 150)` 返回 `{ width: 200, height: 150 }`
- `calculateOutputSize(800, 600, 100, true, 200, undefined)` 返回 `{ width: 200, height: 150 }`（锁定宽高比）
- `getPreviewBlob(file, options)` 返回 Blob，功能与 compressImage 一致

**文件**：`src/utils/imageCompressor.ts`（新建）、`src/utils/imageCompressor.test.ts`（新建）
